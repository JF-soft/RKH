Configuration

Each RKH application must have its own configuration file, called rkhcfg.h. This file adapts and configures RKH by means of compiler definitions and macros allowing to restrict the resources consumed by RKH. Adjusting this definitions allows to reduce the ROM and RAM consumption, and to enhance the system performance in a substantial manner. The "rkhcfg.h" shows the general layout of the rkhcfg.h header file.

Available options:

  • RKH_EN_HCAL

    Enable (1) or disable (0) the state nesting. When RKH_EN_HCAL is set to zero (0) some important features of RKH are not included: state nesting, composite state, history (shallow and deep) pseudostate, entry action, and exit action.
  • RKH_MAX_HCAL_DEPTH

    Determines the maximum number of hierarchical levels. The smaller this number, the lower the static RAM consumption. Typically, the most of hierarchical state machines uses up to 4 levels. Currently RKH_MAX_HCAL_DEPTH cannot exceed 8.
  • RKH_MAX_TR_SEGS

    Determines the maximum number of linked transition segments. The smaller this number, the lower the static RAM consumption. Typically, the most of hierarchical state machines uses up to 4 transition segments. Currently RKH_MAX_TR_SEGS cannot exceed 8.
  • RKH_NUM_STATE_MACHINES

    Number of used state machines.
  • RKH_SIZEOF_EVENT

    Determines the size [in bits] of the RKH event representation RKHE_T. Valid values: 8, 16 or 32. Default 8.
  • RKH_EN_DYNAMIC_EVENT

    Enable (1) or Disable (0) dynamic event support.
  • RKH_EN_DEFERRED_EVENT

    Enable (1) or Disable (0) deferred event support. For using this feature the dynamic event support must be set to one.
  • RKH_EN_PSEUDOSTATE

    Enable (1) or disable (0) the pseudostates usage.
  • RKH_EN_DEEP_HISTORY

    Enable (1) or disable (0) the deep history usage.
  • RKH_EN_SHALLOW_HISTORY

    Enable (1) or disable (0) the shallow history usage.
  • RKH_EN_JUNCTION

    Enable (1) or disable (0) the junction connector usage.
  • RKH_EN_CONDITIONAL

    Enable (1) or disable (0) the conditional connector usage.
  • RKH_EN_INIT_HSM_ARG

    Determines the initialization function prototype of the state machines. When RKH_EN_INIT_HSM_ARG is set to one (1) this function adds as argument a pointer to state machine structure RKH_T. See RKHINIT_T structure definition.
  • RKH_EN_ENT_HSM_ARG

    Determines the function prototype of the state entry. When RKH_EN_ENT_HSM_ARG is set to one (1) this function adds as argument a pointer to state machine structure RKH_T. See RKHENT_T structure definition.
  • RKH_EN_EXT_HSM_ARG

    Determines the function prototype of the state exit. When RKH_EN_EXT_HSM_ARG is set to one (1) this function adds as argument a pointer to state machine structure RKH_T. See RKHEXT_T structure definition.
  • RKH_EN_ACT_HSM_ARG

    Determines the function prototype of the transition action. When RKH_EN_ACT_HSM_ARG is set to one (1) this function adds as argument a pointer to state machine structure RKH_T. See RKHACT_T structure definition.
  • RKH_EN_ACT_EVT_ARG

    Determines the function prototype of the transition action. When RKH_EN_ACT_HSM_ARG is set to one (1) this function adds as argument a pointer to ocurred event. See RKHACT_T structure definition.
  • RKH_EN_GRD_EVT_ARG

    Determines the function prototype of the transition guard. When RKH_EN_GRD_EVT_ARG is set to one (1) this function adds as argument a pointer to ocurred event. See RKHGUARD_T structure definition.
  • RKH_EN_GRD_HSM_ARG

    Determines the function prototype of the transition guard. When RKH_EN_GRD_HSM_ARG is set to one (1) this function adds as argument a pointer to state machine structure RKH_T. See RKHGUARD_T structure definition.
  • RKH_EN_PPRO_HSM_ARG

    Determines the function prototype of the event preprocessor. When RKH_EN_PPRO_HSM_ARG is set to one (1) this function adds as argument a pointer to state machine structure RKH_T. See RKHPPRO_T structure definition.
  • RKH_EN_STATE_NAME

    When RKH_EN_STATE_NAME is set to one (1) the state structure includes its own name as a null-terminated string. When a particular application requires runtime debugging, this option must be enabled. See RKHBASE_T structure definition.
  • RKH_EN_STATE_ID

    When RKH_EN_STATE_ID is set to one (1) the state structure includes an ID number (also called descriptor). This number allows to uniquely identify a state. When a particular application requires runtime debugging (native tracing features), this option must be enabled.
  • RKH_EN_HSM_NAME

    When RKH_EN_HSM_NAME is set to one (1) the state machine structure RKH_T includes its own name as a null-terminated string. When a particular application requires runtime debugging, this option must be enabled.
  • RKH_EN_HSM_ID

    When RKH_EN_HSM_ID is set to one (1) the state machine structure RKH_T includes an ID number (also called descriptor). This number allows to uniquely identify a state. When a particular application requires runtime debugging (native tracing features), this option must be enabled.
  • RKH_EN_HSM_DATA

    When RKH_EN_HSM_DATA is set to one (1) the state machine structure RKH_T allows to reference a data object, which maintains additional information.
  • RKH_EN_PPRO

    When RKH_EN_PPRO is set to one (1) either basic or composite states can use a function to preprocessing the ocurred events.
  • RKH_EN_GET_INFO

    When RKH_EN_GET_INFO is set to one (1) the state machine structure RKH_T includes additional performance information by means of RKH_INFO_T structure.
  • RKH_EN_REENTRANT

    Not yet implemented.
  • RKH_TRACE

    Enable (1) or disable (0) the trace mode. It's described in detail in Debugging tool section.
  • RKH_TRACE_ALL

    Enable (1) or disable (0) all trace points.
  • RKH_EN_EVENT

    If it's enabled (1) records the triggering event.
  • RKH_EN_TRN_SRC

    If it's enabled (1) records the source state of transition.
  • RKH_EN_TRN_TGT

    If it's enabled (1) records the target state of transition.
  • RKH_EN_NXT_STATE

    If it's enabled (1) records the next state of transition.
  • RKH_EN_INT_TRAN

    If it's enabled (1) records the internal transition.
  • RKH_EN_ENTRY

    If it's enabled (1) records the entered state.
  • RKH_EN_EXIT

    If it's enabled (1) records the exited state.
  • RKH_EN_INIT_HSM

    If it's enabled (1) records the initialization process of state machine.
  • RKH_EN_SGT_TGT

    If it's enabled (1) records the target state of transition segment.
  • RKH_EN_RTN_CODE

    If it's enabled (1) records the code returned by rkh_engine() function.
  • RKH_EN_NUM_ENEX

    If it's enabled (1) records the number of entered and exited states.
  • RKH_EN_NUM_ACTSGT

    If it's enabled (1) records the number of transition actions and transition segments.
  • RKH_EN_TIMESTAMP

    When RKH_EN_TIMESTAMP is set to one (1) the trace event structure RKHTREVT_T allows include a timestamp.
  • RKH_SIZEOF_TIMESTAMP

    Determines the size [in bits] of the trace timestamp representation. Valid values: 8, 16 or 32. Default 8.
  • RKH_MAX_NUM_TRACES

    Determines the maximum number of trace events in the stream. The smaller this number, the lower the static RAM consumption.
  • RKH_EN_TRACE_STRING

    When RKH_EN_TRACE_STRING is set to one (1) the trace event structure RKHTREVT_T adds as argument an array of chars to allocate a null-terminated string.
  • RKH_MAX_TRACE_STRING_SIZE

    Determines the size [in bytes] of the trace string argument.
  • RKH_ASSERT

    Enable (1) or Disable (0) assert support.
