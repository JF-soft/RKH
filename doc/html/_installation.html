Installation

< Home

The RKH project is organized through a hierarchical directory structure to facilitate its distribution, support and maintenance. This structure also includes application examples and demos on several platforms.

There are several ways of using RKH, one of them is to use the directory structure as is and the other is to extract the necessary files for including in a particular application.

  • Using files

RKH source files

The Figure 2 RKH source directory shows the \source directory.


Figure 2 RKH source directory 

<RKH-root>              - RKH-root directory
|
+---demo                - RKH demo applications
+---doc                 - RKH documentation
\---source              - RKH source files
|   |   
|   \---include         - RKH platform-independent include files
|   |       rkh.h       - Framework interface
|   |       rkhassert.h - Assert definitions
|   |       rkhitl.h    - Internal use only
|   |       rkhplat.h   - Supported and portable platforms
|   |       rkhtrace.h  - Trace facility
|   |       
|   \---portable        - RKH ports
|   |   \---cws08       - Codewarrior for Freescale S08
|   |       rkhport.h   - RKH platform-dependent include file
|   |   \---lnxgcc      - Linux GCC
|   |       rkhport.h   - RKH platform-dependent include file
|   |   \---vc08        - Visual C++ 2008
|   |       rkhport.h   - RKH platform-dependent include file
|   |                   
|   |   rkh.c           - RKH platform-independent source code.
|   \   rkhtrace.c      - RKH trace facility source code.
|
|   copying.txt         - licence file
|   README              - change log file
\   rkh.chm             - reference manual
  • \include: contains platform-independent header files.
  • \portable: contains platform-independent header files to be used by RKH applications. Each platform-dependent file should be placed in a directory that represents the platform to be used. Each platform directory should be placed within \portable directory, i.e.: \portable\<platform>\rkhport.h. The Figure 2 RKH source directory shows three RKH port directories: \portable\cw08 (Codewarrior for Freescale S08), \portable\lnxgcc (Linux GCC), and \portable\vc08 (Visual C++ 2008).
  • rkh.c: platform-independent source code of RKH. This file requires inclusion of only one platform-specific header file named rkhport.h. It's indirectly included by rkh.h file and it's described in detail in section Porting.
  • rkhtrace.c: platform-independent source code of runtime debugging.

Next, each rkhport.h must be referenced from rkhplat.h header file, located in \include directory. The idea behind conditional compilation is that a rkhport.h can be selectively compiled, depending upon whether a specific value has been defined. The next listing shows the rkhplat.h file according to \portable directory from Figure 2 RKH source directory, where ___CWS08__, ___LNXGCC__, and __VC__ are used to instruct the C/C++ compiler to include header files from the specific RKH port directory.

#if __CWS08__
    #include "..\portable\cw08\rkhport.h"
#elif __VC__
    #include "..\portable\vc08\rkhport.h"
#elif __LNXGCC__
    #include "lnxgcc/rkhport.h" 
#else
    #error "rkhplat.h: Missing platform definition."
#endif

The path of platform-dependent file must be relative.


Demo application

Then, should be created a directory within \demo with the name of application that will make use of RKH. For example: \demo\vc08, as shown in Figure 3 Demo application directories. This directory includes the files needed by the application. Aditionallity, should be created the RKH configuration file rkhcfg.h to be included within the application directory. See Configuration section. This file is mandatory and it's used by RKH.

Using files

Here proposes to extract from RKH directory structure, detailed above, those files required by RKH to use it in a particular application. Therefore, the proposed structure is no longer necessary.

Required source files extracted from \source directory:

rkh.c       - RKH platform-independent source code
rkhtrace.c  - RKH trace tool

Required include files extracted from \include directory:

rkh.h       - Framework interface
rkhassert.h - Assert definitions
rkhitl.h    - Internal use only
rkhplat.h   - Supported and portable platforms
rkhtrace.h  - Trace facility

Required include files extracted from \portable\<platform> directory:

rkhport.h   - RKH platform-dependent include file

The rkhport.h must be referenced from rkhplat.h file as shown below:

#include "rkhport.h"


Application

Next, should be created the configuration file rkhcfg.h to be included within the application directory. See Configuration section for more information. This file is mandatory and it's used by RKH.
